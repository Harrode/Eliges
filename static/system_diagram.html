临床纳排智能检索系统 — 模块架构图

横向布局：Layer 1-5 → 难点 → 解决方案

用户查询："纳入2型糖尿病患者，eGFR<60，排除使用胰岛素的患者"
▼
L1
LLM NLU
自然语言理解
功能
  • 解析纳入/排除条件
  • 提取诊断、药物、检验值
  • 识别性别、年龄、时间
难点
  • inclusion/exclusion 易混
  • 逻辑结构识别
  • 检验值格式多样
解决方案
  • LLM 模板填充
  • 后处理标准化
  • 排除词上下文检测
  • 检验值标准化
L2
检索构建
条件→查询
功能
  • 字段映射
  • 查询类型选择
  • 两阶段检索策略
难点
  • LLM 直接查库不可控
  • 字段名不同
  • 嵌套查询构建
解决方案
  • Semantic View 安全层
  • 字段映射配置
  • Two-stage 检索
L3
全文检索
ES 检索引擎
功能
  • 全文检索 + 模糊匹配
  • 检验值区间过滤
  • 日期范围过滤
难点
  • 中英文混合检索
  • 检验值区间匹配
  • 时间条件过滤
解决方案
  • ICU 分词器
  • fuzziness: AUTO
  • 嵌套查询 + 值范围
  • 日期范围过滤
L4
规则过滤
硬约束校验
功能
  • 诊断匹配验证
  • 检验值精确比较
  • 排除条件验证
  • 命中依据生成
难点
  • 同义词（肝病↔肝硬化）
  • 操作符兼容
  • 否定处理
  • 结果不足处理
解决方案
  • 同义词映射
  • 操作符兼容匹配
  • 否定上下文检测
  • Loop-back 放宽重试
L5
LLM 解读
分析与问答
功能
  • 生成分析报告
  • 交互问答
  • 结果解释
难点
  • 长文本理解
  • 多轮对话
  • 结果准确性
解决方案
  • 上下文注入
  • Markdown 格式化
  • 基于检索结果问答
⟲ Loop-back：结果不足时放宽条件，反馈给 L1 重新解析
▼
检索结果：患者列表 + 匹配依据 + 分析报告

数据流

用户查询 → L1 LLM NLU → 条件JSON → L2 检索构建 → ES查询 → L3 全文检索 → 候选患者 → L4 规则过滤 → 精确结果 → L5 LLM解读 → 返回用户

Layer核心挑战解决方案
